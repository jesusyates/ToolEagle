# V104.1 — SEO Content Safety Layer

**Scope**: Programmatic pSEO (`ProgrammaticPageTemplate`, keyword sync), **not** the Content Safety Engine for AI tool APIs (see `docs/MEMORY.md` 「二点五」).

## What shipped

1. **Prompt / copy rules** — `SEO_CONTENT_GENERATION_RULES` in `src/lib/seo/pseo-content-safety.ts` (maintainer reference; EN-oriented bullets).
2. **Tone** — Examples and prompts rebuilt per locale: expert, neutral, advisory; Chinese uses **「提升内容表现的方法」**-style framing where a generic “技巧” label appears; removed comment-bait / hype phrasing from EN templates.
3. **Compliance buffer** — Short lines before blockquotes: CN **「建议根据平台规则调整。」「不同账号情况可能不同。」**; EN equivalents for `en` locale.
4. **Quality** — `evaluatePseoSyntheticContentQuality` flags thin keywords, repetitive templates, generic AI phrasing; `combinedProgrammaticQualityScore` = `min(heuristicQualityScore, synthetic score)` applied in `syncSeoKeywordsFromSignals` for `seo_keywords.quality_score`.
5. **Internal review signal** — `inferHighRiskPseoTopic(keyword)` (medical/finance/adult/中文灰产类启发式); exposed in `GET /api/seo/quality-check` as `highRiskTopicKeywordCount` + `highRiskTopicSlugsSample`. No UI flag on pSEO pages.

## Explicitly not built (per spec)

- No 🛡️ trust-signal UI on pSEO pages.
- No aggressive mid-page marketing on pSEO: `SeoMidUpgradeCta` uses `tone="pseo"` from `ProgrammaticPageTemplate`.
