# V109.4 — 英文站导航收口、主路径与 CTA 规范

在 **V109.3 首页模块顺序不变**（Hero → Quick tools → Examples → ValueProof → Trending → …）前提下，减少顶栏混乱、统一主路径与 CTA 语气，并对首页工具区做 **主次分层**。

---

## 1. 英文站导航：修改前问题清单

| 现象 | 说明 |
|------|------|
| **顶栏入口过多（14+）** | Home、Discover、AI Prompts、Learn AI、Answers、Creators、Community、Favorites、Me、Dashboard、Creator、Pricing、Blog、About 与 Logo 抢同一认知层级。 |
| **语义重复** | Discover / Learn AI / Blog / Answers 均偏「内容发现」，新用户不知先点谁；AI Prompts 与 Learn AI 均偏「提示词/学习」。 |
| **与「工具站」主线竞争** | Dashboard、Creator、Favorites、Me 偏已登录用户，却与首次访问主路径同级。 |
| **Tools 缺失** | 无直接 **Tools** 索引链接，「免费工具」依赖记忆或页内跳转。 |
| **Guides 边界不清** | How-to（`/en/how-to`）、blog、discover 未在顶栏收敛为一个「指南/玩法」入口。 |
| **Upgrade 与 Pricing** | 二者不同（Pro vs 定价页），保留；与 Market 切换并列合理。 |

**结论**：顶栏只做 **Tools / Guides（EN how-to）/ Pricing** + Market + Pro + Auth；其余下沉 **Footer**（原链接未删路由）。

## 2. 修改后的导航结构

| 元素 | 修改前 | 修改后 |
|------|--------|--------|
| 主导航 | 14 个 `TranslateAwareLink` | **3 个**：`/tools`、`/en/how-to`、`/pricing` |
| 文案键 | `nav.home` … `nav.about` 混排 | `nav.tools`、`nav.guides`、`nav.pricing` |
| Logo | 回首页 | 不变 |
| Market | `MarketSelector` | 不变 |
| Pro | `UpgradeLink` | 不变 |
| 账户 | `AuthButton` | 不变 |
| 移动端 | 曾 `hidden sm:flex` 导致小屏无主导航 | **始终显示** 3 链（`text-xs` 适配） |

**未改**：`layout.tsx`、`SiteFooter` 路由集合（Footer 仍提供 Blog、About、Learn AI、Dashboard 等）。

## 3. 英文站默认主用户路径（推荐）

```
首页（价值 + 主 CTA）
  → /tools/ai-caption-generator（生成发布包）
  → 结果页 / 复制 / 再用
  → 需要升级时：Pricing（/pricing）或 Pro CTA
  → 需要学法时：Guides（/en/how-to）
```

**首次用户默认路径**：**首页 → Post Package（`ai-caption-generator`）→ 再按需打开 Caption / Hook / Script / Idea**（在首页「More generators」中）。

---

## 4. CTA 文案规范摘要（英文站）

| 层级 | 用法 | 示例 |
|------|------|------|
| **主转化（生成）** | 动词 **Generate** + 具体对象 | `Generate a post package`（首页 Hero + 主工具卡） |
| **次转化（浏览）** | **Browse** + 范围 | `Browse tools`（首页次按钮） |
| **次级工具入口** | **Open tool**（与主生成区分，避免五个全是「Generate」） | More generators 四卡 |
| **信任** | 短句、可验证 | `Free to try · No signup required` |
| **升级** | 保持现有 Pro 文案 | `Grow faster with Pro` 等 |

**避免**：同一漏斗里混用无层级区别的「Try free」「Start now」「Go」与主按钮并列抢词。

---

## 5. 首页后半段模块：保留 / 下沉建议（未大删）

当前顺序：**Hero → Quick tools → Examples → ValueProof → Trending（EN）→ Platform → Popular → Why → Blog → CreatorModeDemo**

| 模块 | 建议 | 理由 |
|------|------|------|
| Trending（EN guides） | **保留**，位置可继续偏后 | SEO + 发现；已 EN-only |
| Platform tools（三平台） | **可下沉** 至 `/tools` 或 Discover 类页强化 | 与 Quick tools 部分重叠 |
| Popular tools（ToolCard 网格） | **可下沉** 或缩为 3 卡 | 与 Quick tools 重叠 |
| Why ToolEagle | **保留短文** 或合并进 Footer | 信任背书 |
| Blog | **保留链接级**；首页可只保留 2 篇 | 减长度 |
| CreatorModeDemo | **最易移出首页** | 非首次用户主路径；可链到 `/creator` |

**原则**：前半段 = **开始用工具**；后半段 = **信任 + 轻量发现**；大发现交给 Tools 索引与 Guides。

---

## 6. 五个首页主工具：主次关系（产品定义）

| 层级 | 工具 | 角色 |
|------|------|------|
| **默认 / 新手入口** | **Post Package**（`ai-caption-generator`） | 一条龙产出；与 Hero、主卡 CTA 一致。 |
| **快速单点** | **Caption**（`tiktok-caption-generator`） | 只要文案、已有选题。 |
| **注意力** | **Hook** | 只要开头。 |
| **结构进阶** | **Script** | 要分段与口播结构。 |
| **选题扩展** | **Idea** | 要方向与角度。 |

**实现**：首页 **单独一块「Recommended」主卡**（sky 强调 CTA：`Generate a post package`）；**「More generators」** 四卡使用 **outline 按钮 `Open tool`**，避免五卡同级黑色「Generate」。
