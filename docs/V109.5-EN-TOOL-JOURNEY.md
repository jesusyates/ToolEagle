# V109.5 — 英文主工具页主路径、文案与 Next steps

在 **V109.3 / V109.4** 不动首页与顶栏结构的前提下，统一 **5 个主工具页** 的首屏说明、主按钮文案与结果后的 **Next steps**；**未改** `layout`、路由、metadata、sitemap。

---

## 1. 五个主工具页角色定义

| 工具 | Slug | 角色 | 说明 |
|------|------|------|------|
| **Post Package（默认入口）** | `ai-caption-generator` | **default** | 一条龙发布包；**新手默认路径**；canonical **`/ai-caption-generator`**。 |
| **Caption** | `tiktok-caption-generator` | **quick_single** | 偏 TikTok/Reels 单平台整包，快速产出。 |
| **Hook** | `hook-generator` | **attention** | 只要开头、注意力。 |
| **Script** | `tiktok-script-generator` | **structure** | 分段与口播结构。 |
| **Idea** | `tiktok-idea-generator` | **angles** | 选题与角度，偏「破冰」。 |

配置见 **`src/config/en-tool-journey.ts`**（`EN_TOOL_JOURNEY`）。

---

## 2. 调整了哪些工具页首屏逻辑

| 范围 | 改动 |
|------|------|
| **共用壳 `ToolPageShell`** | 可选 **`introProblem` / `introAudience`**（英文全球站）：首屏在标题与描述下增加 **What it solves / Best for** 灰底说明块。 |
| **`PostPackageToolClient`** | 全球英文（`siteMode !== china`）时从 `getEnToolJourney(toolSlug)` 注入上述字段；主按钮文案优先用 journey 的 **`generateCta`**。 |
| **`GenericToolClient`** | 仅 **`locale === "en"`** 且 slug 在 journey 表中时注入 intro + **`generateCta`** 覆盖 `generators` 的 `buttonLabel`。 |
| **首页 → 主工具** | Hero 与 Quick tools 主卡 **`/ai-caption-generator`**（与 canonical 一致，不再经 `/tools/ai-caption-generator` 跳转）。 |
| **`/ai-caption-generator` 页面** | `generateButtonLabel` 与 journey 对齐为 **「Generate a post package」**。 |

未改：各页 **metadata**、**路由文件**；抖音 / 中文 `siteMode=china` **不展示** EN intro / Next steps（沿用原逻辑）。

---

## 3. CTA 文案统一规则（英文）

| 类型 | 规则 | 示例 |
|------|------|------|
| **主路径 · 发布包类** | `Generate` + 具体对象 | `Generate a post package` / `Generate post packages` |
| **辅助 · 单点输出** | `Generate` + 名词 | `Generate hooks` / `Generate script` / `Generate ideas` |
| **首页 / 卡片区** | 与 **`en-tool-journey`** / **`home.ctaGeneratePackage`** 一致 | 见 V109.3/V109.4 |
| **次动作** | `Browse tools`、Next steps 用 **说明性短语 + 箭头** | 见 `ToolNextSteps` |

原则：真实、克制、结果导向；避免同屏多个无层级区别的「Try / Start / Go」。

---

## 4. 结果区优化说明

- **Post Package / TikTok package / Hook**：结果仍由 **`PostPackageResults`** 渲染（分段、锁定预览、复制等），**未做大改**；V109.5 在结果块 **下方** 增加 **`ToolNextSteps`**，强化「用完再去哪」。
- **Idea / Script（Generic）**：结果仍为 **`ToolResultListCard`** 列表 + 复制；同上在列表下增加 **Next steps**。

若需进一步「卡片化」大段纯文本，可单列后续迭代（本版不扩 scope）。

---

## 5. 统一 Next steps 组件

- **新增**：`src/components/tools/ToolNextSteps.tsx`
- **行为**：`hasOutput === true` 时在结果区下方展示 **2 条链式建议**（来自 `EN_TOOL_JOURNEY[slug].nextSteps`）。
- **覆盖**：`PostPackageToolClient`（全球 EN）+ `GenericToolClient`（`locale === "en"` 且 slug 有 journey）。
