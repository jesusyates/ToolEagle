# Blueprint Usage Note

Primary decision anchor: `docs/system-blueprint.md`.

Rules:

1. Blueprint is the system-level decision anchor for growth, structure, and anti-drift.
2. If implementation conflicts with blueprint rules, blueprint wins and implementation must be corrected.
3. Update the blueprint only when system phase truly changes (not for short-term tactical noise).
4. Do not invent new page roles or SEO control logic outside blueprint definitions.
5. Use `generated/system-blueprint.json` only as a machine-readable companion; markdown remains source of truth.
