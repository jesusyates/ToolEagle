# memory/ — 运行手册库（怎么做）

**这不是「规则正文」。** 产品方向、合规、架构原则 **只以** 仓库根目录相对路径 **`docs/MEMORY.md`** 为准。本目录只放 **执行步骤、清单、协作说明**，方便人和 AI 按任务打开对应文件。

**全链路执行（推荐 Cursor 在相关任务时阅读）：** `tooleagle-project-operating-manual.md` — 日常生产（含 V154/V167）、上传、检索数据集与飞轮（V164–V168）、Vercel 构建纪律、**运行时真相**与能力状态表、变现、风控、治理与汇报格式；与 MEMORY 冲突时以 MEMORY 为准。

## 给 ChatGPT：完整规范（请优先读这个）

→ **`memory/FOR-EXTERNAL-AI-CHATGPT.md`**  

内含：任务边界、**文件名与格式**、步骤怎么写、**交付物清单**、以及一段 **可直接复制给 ChatGPT 的开场指令**。把该文件全文或路径交给 ChatGPT 即可。

## 给协作者：极简备忘

1. **一事一文件或一流程一文件**，文件名用英文 kebab-case，例如 `seo-publish-queue-steps.md`。
2. 每个文件建议包含（可直接复制 `memory/_TEMPLATE-RUNBOOK.md`）：
   - **目的**（1～3 句）
   - **前置条件**（环境、分支、密钥、依赖脚本）
   - **步骤**（编号列表，每步写清：在哪运行、命令或要点、成功判据）
   - **常见坑 / 回滚**
   - **相关代码或配置路径**（可点击的仓库内路径）
3. **不要**在本目录重复抄写 MEMORY 里的原则；若某步依赖规则，写一句「见 `docs/MEMORY.md` § xxx」即可。
4. 写完或重大更新后，在 **下面「索引」** 里加一行链接与一句话摘要。

## AI 助手怎么用（重要）

- **需要哪里就自己读哪里**：助手应**主动** Read 本目录文件，**不要**要求用户把手册全文贴进对话；用户描述任务即可。
- **不会**在单次对话里「无差别全文读取」整个 `memory/`（token 与噪音）。做法是：**先 Read 本 `README.md` 看索引** → 必要时 **搜索 `memory/`** → 再 **Read 与当前任务相关的少数文件**（通常 1～3 个）。
- **规范冲突时**：一律 **`docs/MEMORY.md` 优先**，本目录仅作操作参考。
